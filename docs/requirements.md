# 日曜学校 出席管理アプリ 要件定義（MVP）

## 概要

教会の日曜学校における出席管理を行う Web アプリ。
教師が毎週の出席を入力し、クラス別および全体の集計を行う。
教師数は約 10 名。

## 技術スタック

- フロント / バックエンド: Next.js
- DB: Neon (PostgreSQL)
- ORM: Drizzle
- 認証: Clerk（メール OTP 認証）
- メール送信: アプリ側（Cron + 外部メールサービス想定）

## 開発・DB運用方針（MVP）

- ローカル開発時も Next.js アプリから Neon の開発用 DB に接続する
- 本番用 DB と開発用 DB は分離する
- 接続情報は環境変数 `DATABASE_URL` で管理する
- スキーマ変更は Drizzle の migration で管理する
- 初期データは seed スクリプトで投入する

## ユーザー

### 教師

- Clerk のメール OTP でログイン
- 担当クラスの出席入力
- 出席履歴の確認

### 管理者（admin）

- 教師の権限を持つ + 管理機能
- クラス管理
- 年度管理
- 教師へのクラス割当

## ドメインモデル

### 学年

- 幼稚園 / 保育園
- 小学 1〜6 年
- 中学 1〜3 年

### 年度（`school_year`）

- 出席は年度単位で管理
- 年度ごとにクラス・所属を持つ

### クラス（`class`）

- 学年に紐づく
- 年度ごとに存在
- 教師が担当

### 生徒（`student`）

- 名前
- 現在の学年
- 年度ごとにクラスに所属

### 出席（`attendance`）

- 毎週日曜単位で記録
- 生徒ごとに出席状態を保持

## データモデル（概要）

### `teachers`

- `id`
- `email`
- `role` (`admin` / `teacher`)
- `active`

### `school_years`

- `id`
- `label`（例: `2026年度`）
- `start_date`
- `end_date`
- `is_active`

### `classes`

- `id`
- `name`
- `grade_code`
- `school_year_id`
- `teacher_id`

### `students`

- `id`
- `name`
- `current_grade_code`
- `active`

### `student_class_assignments`

- `id`
- `student_id`
- `school_year_id`
- `class_id`
- `assignment_type` (`auto` / `manual`)

### `attendance_dates`

- `id`
- `school_year_id`
- `date`（日曜）

### `attendance_records`

- `id`
- `attendance_date_id`
- `student_id`
- `status`（出席 / 欠席 など）
- `note`

## クラス割当ルール

- 基本: 学年に応じて自動割当
- 例外: 手動でクラス変更可能
- 年度ごとに所属を管理

## 機能一覧

### 認証

- Clerk のメール OTP でログイン
- メールアドレスでユーザー識別
- `role` による権限制御

### 出席管理

- 毎週日曜ごとに出席入力
- クラス単位で一覧入力
- 過去の出席の閲覧・編集可能

### 生徒管理

- 生徒の登録 / 更新 / 削除
- 学年の設定
- 年度ごとのクラス所属管理

### クラス管理（admin）

- クラスの登録 / 更新 / 削除
- 学年との紐付け
- 教師の割当

### 年度管理（admin）

- 年度の作成 / 切替
- 新年度作成時の処理
- 学年の繰り上げ
- クラス自動割当
- 必要に応じて手動調整

### 集計

- クラス別出席率
- 全体出席率
- 週 / 年度単位の集計

### メール通知（予定）

- 毎週自動送信（Cron）
- 内容（未確定）
  - 出席入力リマインド
  - 集計結果通知 など

## 非機能要件

### セキュリティ

- Clerk による認証
- admin 機能は `role` で制御

### 運用

- 小規模（教師約 10 名）
- 個人情報は最小限（名前・学年のみ）

### 保守性

- 年度ごとのデータ分離
- 将来的に認証や通知の拡張が可能な構造

## 未決事項 / 要確認

- メール送信の対象（教師 / 管理者 / 保護者）
- 出席ステータスの種類（出席 / 欠席以外が必要か）
- 年度開始月（4月 or 教会独自）
- 教師の複数クラス担当可否
- 出席入力の締切・編集ルール
- 集計の粒度（週 / 月 / 年 / 学年別など）
- 生徒識別に追加情報が必要か（ふりがな等）
